---
title: Troubleshooting
description: How to troubleshoot issues encountered with Ollama
---

Sometimes Ollama may not perform as expected. The fastest way to understand what happened is to inspect the local Windows logs. Open these locations with `<Win>+R`:

- `explorer %LOCALAPPDATA%\Ollama` to view logs. The most recent logs are typically `app.log`, `server.log`, and `upgrade.log`.
- `explorer %LOCALAPPDATA%\Programs\Ollama` to browse the installed desktop app binaries.
- `explorer %HOMEPATH%\.ollama` to browse models and configuration.
- `explorer %TEMP%` to inspect temporary executable files in one or more `ollama*` directories.

To enable additional debug logging, first quit the running app from the tray menu, then launch it from PowerShell:

```powershell
$env:OLLAMA_DEBUG="1"
& "$env:LOCALAPPDATA\Programs\Ollama\ollama app.exe"
```

Join the [Discord](https://discord.gg/ollama) for help interpreting the logs.

## LLM libraries

Ollama includes multiple LLM libraries compiled for different GPUs and CPU vector features. Ollama tries to pick the best one based on the capabilities of your system. If this autodetection has problems, or you run into GPU crashes, you can temporarily force a specific library.

In the server log, you will see a message that looks something like this (varies from release to release):

```
Dynamic LLM libraries [rocm_v6 cpu cpu_avx cpu_avx2 cuda_v11 rocm_v5]
```

For a quick Windows test session, set `OLLAMA_LLM_LIBRARY` before launching the app:

```powershell
$env:OLLAMA_LLM_LIBRARY="cpu_avx2"
& "$env:LOCALAPPDATA\Programs\Ollama\ollama app.exe"
```

For a persistent configuration, add the variable in Windows environment variables and restart the app.

## GPU Discovery

When Ollama starts up, it takes inventory of the GPUs present in the system to determine compatibility and how much VRAM is available. Sometimes this discovery can fail to find your GPUs. In general, running the latest driver will yield the best results.

If you are experiencing problems getting Ollama to correctly discover or use your GPU for inference, the following may help isolate the failure:

- Update your NVIDIA or AMD drivers to the latest supported version.
- Restart the app after changing drivers or environment variables.
- Enable `OLLAMA_DEBUG=1` and inspect `server.log` for `GPU discovery`, `Dynamic LLM libraries`, or backend-selection messages.
- For NVIDIA diagnostics, you can also set `CUDA_ERROR_LEVEL=50` before launching the app.
- For AMD diagnostics, you can also set `AMD_LOG_LEVEL=3` before launching the app.

## Windows Terminal Errors

Older versions of Windows 10 (e.g., 21H1) are known to have a bug where the standard terminal program does not display control characters correctly. This can result in a long string of strings like `←[?25h←[?25l` being displayed, sometimes erroring with `The parameter is incorrect` To resolve this problem, please update to Win 10 22H1 or newer.
